# DEPLOYMENT TAOBAO SELLER NOTES

本页面向“淘宝商家客服场景”的部署与运营注意事项，重点覆盖高频售后、合规话术和成本控制。

## 1. 业务配置优先于模型自由生成

- 把退货、换货、补发、发票、物流时效等规则固化为知识库文本，并定期更新。
- 对价格、优惠、赔付标准等敏感问题，优先走规则回复或人工确认。
- 禁止模型对库存、赔付金额、到账时效做无依据承诺。

## 2. 话术与合规

- 避免绝对化表达（如“全网最低”“百分百”）。
- 避免引导站外交易或索要与售后无关敏感信息。
- 保存客服回复日志，支持运营复盘和争议追溯。

## 3. 数据安全与最小化采集

- 对手机号、地址、订单号做脱敏展示。
- 仅保存排障所需字段，不在日志中记录明文密钥。
- 设置聊天与上传数据保留周期，超期自动清理。

## 4. 成本与稳定性

- 为 LLM 请求设置超时（`ASSISTGEN_LLM_TIMEOUT_SECONDS`）和重试上限。
- 建议增加限流策略，避免大促期间请求风暴导致成本失控。
- 准备降级方案：模型超时时切换模板回复或人工接管。

## 5. 多店铺隔离建议

- 按店铺维度隔离会话和知识库数据（建议后续增加 `shop_id` 字段）。
- 不同运营角色分配不同权限，避免跨店误操作。
- 发布前在测试店进行灰度，确认再扩展到主店。

- [ ] 已验证 `GET /health` 与 `POST /api/chat`
- [ ] 已确认密钥仅通过环境变量注入
- [ ] 已确认知识库内容和店铺最新规则一致
- [ ] 已准备客服人工接管流程与异常应答模板
